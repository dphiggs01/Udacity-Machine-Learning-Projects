Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer:

One of the principal benefits of splitting data into testing and training sets is that after we have developed a model we want to see how accurate it will be on new unseen data. To do this, we hold out some of our dataset for testing only. In this way, the testing data will not have been seen by our model, and the results of the tests will provide a good Indication if our model is behaving well on unseen data.

Additionally, in machine learning two common errors occur during model development underfitting the data which is an oversimplification of the problem and overfitting the data which provides a complex model to precisely fit the test data but does not generalize well to new unseen data.

Appropriately splitting the dataset into training and testing sets can help minimizes the likelihood of introducing these problems into our predictive model.

For example, if we use the full dataset for both training and testing, we would likely overfit our model to the data and have a hard time generalizing to new unseen data. On the other hand, if the split data provides a poor representation of overall dataset (say from a sample that is too small), we may underfit the data and also not be able to predict well.

To help mitigate both of these potential problems we could apply k-fold cross-validation to our dataset. The cross-validation would randomly split the data into k subsets. Then the model would be trained on all the other subsets and tested on a holdout. This would occur k times ensuring that we tested on the holdout data only once for each of the k subsets. Once complete the results could be averaged.

What is the k-fold cross-validation training technique?

K-fold cross-validation training splits the training dataset into K subsets (or folds) of equal size. And for each of the k folds we hold out one of the subsets for testing, we then train on the remaining subsets. We do this once for each of the k folds until we have tested on all the data. We then calculate the average performance of our model over all the folds.

What benefit does this technique provide for grid search when optimizing a model?

When using a single validation set, we may overfit our parameter choices to the specific single set. However, when we use cross-validation, we have k sets of data that we can test and validate against increasing our chances to generalize better.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)

    # Create a decision tree regressor object
    regressor = DecisionTreeRegressor(random_state=0)

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':range(1,11)}

    # Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Answer:

The current model although interesting and could potentially be used to facilitate demonstrating market indicators is not a useful tool for actually predicting current home prices. There are numerous reasons for not being confident in the predicted results.

First, the collected data is not likely representative of current Boston market conditions as it was collected in 1978. And even if the median prices have been modified for to support 35 years of market inflation the other feature data is decades out of date.

Second, even though we would like to create the simplest model possible that accurately fits the data, it is unlikely that the limited feature set provided could cover the many nuances in the Boston housing market. Our simple model does not even include lot size or taxes as a consideration.

Third, if we take into consideration, the predicted trial runs we can see a $20k standard deviation in prices over ten test runs indicating a significate amount of variance which may suggest that the model is underfitting the data.

Finally, we would not want to use this model for other areas either urban or rural as the location of a property is one of the most important indicators of its value and thus we need localized and timely data to create an accurate model.
